🔍 SEO Optimization

Lila provides powerful built-in tools to manage Search Engine Optimization (SEO) metadata, automatically inject it into your templates, and generate necessary files like sitemap.xml.

1. The @seo Decorator

The easiest way to define metadata for a specific route is using the @seo decorator. It automatically injects the data into the request.state.seo object.

Using @seo 

from lila.core.routing import seo

# Basic usage
@router.get("/")
@seo(title="Home Page", description="Welcome to our site")
async def index(request):
    return render(request, "index")

# Multilingual support (Dictionary)
@router.get("/about")
@seo(title={"en": "About Us", "es": "Sobre Nosotros"})
async def about(request):
    return render(request, "about")

# Integration with translations (uses locales/translations.json)
@router.get("/contact")
@seo(title="translate:contact_title")
async def contact(request):
    return render(request, "contact")

2. Centralized SEO Configuration

If you prefer to keep your routes clean, you can define all your SEO metadata in a centralized file located at app/seo.py. Lila will automatically load it.

app/seo.py 

# Map route paths to their SEO metadata
SEO_CONFIG = {
    "/": {
        "title": "Global Home Page",
        "description": "This applies to the root path automatically."
    },
    "/services": {
        "title": {"en": "Services", "es": "Servicios"}
    }
}

3. Template Integration

Lila automatically injects the resolved SEO object into your Jinja2 template context. You can access it directly in your base.html. It also works automatically with renderReact for Single Page Applications.

templates/html/lila/base.html 

<head>
    <!-- Use the seo object injected by the Router -->
    <title>{{ seo.title | default(title) }}</title>
    <meta name="description" content="{{ seo.description | default(description) }}">
    <meta name="keywords" content="{{ seo.keywords | default(keywords) }}">
</head>

4. Generating Sitemap & Robots

Lila provides CLI commands to automatically generate your sitemap.xml and robots.txt files based on your registered routes. It intelligently ignores dynamic paths (like /user/{id}) and private areas (like /admin).

CLI 

# Generate sitemap.xml in the public/ folder
lila-seo sitemap --domain https://mywebsite.com

# Generate robots.txt in the public/ folder
lila-seo robots --domain https://mywebsite.com
