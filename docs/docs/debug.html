Debug & Profiling

Lila Framework includes an advanced DebugMiddleware that automatically monitors the performance of your application. When debugging is enabled, it tracks RAM usage, CPU consumption, execution time, and more for every HTTP request.

Enabling Debug Mode & Dashboard

The debugging and profiling features are only active when your application is in debug mode (DEBUG=True in your .env or config).

In addition, the Debug Dashboard (/debug) is disabled by default to keep the framework extremely lightweight. To enable it, pass debug_html=True during the App initialization in main.py:

main.py 

from lila.core.app import App

app = App(
    debug=True,
    debug_html=True,  # Enables the /debug dashboard panel
    # ...
)

What gets tracked?

For every non-static asset request, the middleware logs the following data into a local SQLite database (`debug` table):

  • Path & Method: Endpoint accessed and HTTP method.
  • IP Address: Client's IP.
  • RAM Used: Memory allocated during the request lifecycle.
  • CPU: Processor time consumed.
  • Execution Time: Total time taken to respond.
  • Status Code: The HTTP response status.

The Debug Dashboard

Lila automatically mounts a visual dashboard for you to inspect these logs without needing any additional setup. Simply navigate to:

http://localhost:8000/debug/

Here you will see a real-time table with all the recorded requests and their performance metrics.

Fetching Data Programmatically

If you want to fetch this data programmatically, you can hit the same endpoint with a `fetch` query parameter:

Terminal 

curl http://localhost:8000/debug/?fetch=True

Clearing Logs

To clean up your database and remove all previous debug logs, simply send a DELETE request to the dashboard URL:

Terminal 

curl -X DELETE http://localhost:8000/debug/
Note: The /debug endpoint and the DebugMiddleware are completely disabled when DEBUG=False to ensure zero performance overhead in production environments.
