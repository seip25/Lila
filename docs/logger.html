Logger Utility

Lila provides a built-in Logger utility specifically tailored for web applications. It handles the formatting, storing, and rotation of log files automatically, while actively scrubbing sensitive data (like passwords) from recorded requests.

Basic Logging

You can import the Logger anywhere in your application to record info, warnings, or errors.

app.py 

from lila.core.logger import Logger

# Standard logging
Logger.info("Application started successfully.")
Logger.warning("Database connection is slow.")

# Logging errors with tracebacks
try:
    1 / 0
except Exception as e:
    # Passing the exception object automatically logs the traceback
    Logger.error("A critical error occurred", exception=e)

# Custom log types
Logger.log("audit", "User modified their profile")

Logging Requests Securely

The Logger.request() method is an asynchronous helper designed to record incoming requests. It automatically logs the IP, Operating System, HTTP Method, Path, and Body.

Security First: It actively scrubs any JSON or form keys named password, password_2, or token to prevent sensitive data from leaking into your plain text log files.

auth.py 

from lila.core.logger import Logger
from lila.core.request import Request

async def secure_endpoint(request: Request):
    # Generates a safe string representing the request
    log_string = await Logger.request(request)
    
    Logger.info(f"Incoming Request: {log_string}")

Log Storage and Rotation

Logs are saved into the directory defined by PATH_LOG_BASE_DIR in your app/config.py. Lila organizes logs into folders by date (e.g., DD-MM-YYYY/info.log).

To prevent your server's disk from filling up, the module includes a helper function delete_old_logs(days). By default, it deletes folders older than 30 days. You can set this up as a background task or a cron job.

cron.py 

from lila.core.logger import delete_old_logs

# Delete logs older than 15 days
delete_old_logs(days=15)
