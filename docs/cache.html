Memory Cache

Lila includes a simple but powerful in-memory caching system to reduce database load and improve response times. It supports TTL (Time To Live), allowing items to expire automatically.

Basic Usage

You can use the Cache class from lila.core.cache to store and retrieve any Python object.

Using Cache in Code 

from lila.core.cache import Cache

# Store a value for 5 minutes (300 seconds)
Cache.set("my_key", {"data": "value"}, ttl=300)

# Retrieve a value
data = Cache.get("my_key")

if data:
    print("Value found in cache")
else:
    print("Value expired or not found")

# Delete a specific key
Cache.delete("my_key")

# Clear everything
Cache.clear()

Automatic Route Caching

The most common way to use caching is through the Router. By default, all GET routes have a 30-second cache.

For more details on how to configure route caching, see the Routes Documentation.

️ Core Optimizations

Lila uses this caching system internally to optimize several core features:

  • OpenAPI: The generated openapi.json is cached for 1 hour.
  • Vite: Asset detection is cached to avoid constant disk I/O during development.
  • Translations: Localization files are loaded once and kept in memory.

Warning: The memory cache is volatile. If you restart the server, all cached data will be lost. It is intended for temporary data and performance optimization, not for persistent storage.
